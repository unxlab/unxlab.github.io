Centos 7搭建Gitlab服务器

###GitLab简介：

GitLab 是一个用于仓库管理系统的开源项目，使用Git作为代码管理工具，并在此基础上搭建起来的web服务。安装方法是参考GitLab在GitHub上的Wiki页面。
GitLab是由GitLabInc.开发，使用MIT许可证的基于网络的Git仓库管理工具，且具有wiki和issue跟踪功能。使用Git作为代码管理工具，并在此基础上搭建起来的web服务。
由乌克兰程序员DmitriyZaporozhets和ValerySizov开发，它由Ruby写成。后来，一些部分用Go语言重写。截止2018年5月，该公司约有290名团队成员，以及2000多名开源贡献者。GitLab被IBM，Sony，JülichResearchCenter，NASA，Alibaba，Invincea，O’ReillyMedia，Leibniz-Rechenzentrum(LRZ)，CERN，SpaceX等组织使用。

###Why GitLab？

提供Git项目仓库托管服务的是业界闻名的GitHub，但你要将代码上传到GitHub上，而且将项目设为私有是需要收费的。而GitLab则是开源免费的（社区版免费，企业版需订阅）。同样是采用Ruby on Rails开发，可以让你在自己的内网搭建一个“高仿”版的GitHub。企业的代码管理使用GitLab还是非常诱人的。
所以本章就GitLab的搭建开一篇，因为同类型的文章网络上数不胜数，造个轮再让自己加深印象也是可以的。

###安装前提：
GitLab支持在多系统下进行部署：

  • Ubuntu
  • Debian
  • CentOS
  • openSUSE
  • Red Hat Enterprise Linux (please use the CentOS packages and instructions)
  • Scientific Linux (please use the CentOS packages and instructions)
  • Oracle Linux (please use the CentOS packages and instructions)
  • #####但是不支持以下系统部署GitLab：

Non-Unix operating systems such as Windows
可以看到官方是不支持将来也不会考虑支持的安装平台就是Windows了，如果只有windows，那么可以考虑搞个虚拟机来安装Linux支持版本，再使用GitLab，挺任性。

#####硬件需求：
######CPU：
官方建议双核cpu，最多支持500用户。1个核心只支持100用户，同时也会导致程序运行缓慢，这个就按照官方建议来操作。

######内存：
8G，最多支持100个用户。同样按照官方建议来进行配置，因为4G的内存可能会导致配置期间发生奇怪的错误，有兴趣的可以研究一下。本章节选择8G内存来进行部署。

######交换空间（Swap）：
至少2G。官方文档提示即使有足够的可用内存，但是足够的交换空间可以在你的内存发生改变时减少发生错误的可能性。

######存储：
够用就行，本章使用的存储为400G，应该足以使用。

######数据库：
PostgreSQL，（version 9.6）这是官方强烈推荐的数据库首选，非常不推荐的是MySQL/MariaDB，因为大部分的GitLab功能在这个数据库上不支持。同时还建议正在使用Mysql／MariaDB的用户尽快迁移到PostgreSQL。

####部署环境：
本次GitLab要部署在CentOS 7上。发行版本是CentOS Linux release 7.6.1810 (Core)
内存8G
交换空间8G
存储空间400G
数据库PostgreSQL

以上是准备好的安装环境，基本符合GitLab官方推荐需求了。

另一种就是手动安装，把代码下载下来，然后安装各种环境，数据库，编译环境等。
一步步安装起来，虽然比较麻烦，但是可以安装各种分支版本。比如中文翻译版。

先按照官方推荐版本来进行安装。
